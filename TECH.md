# Описание функционала

## Назначение

Программа работает только в Windows и помогает обрабатывать выделенный пользователем текст через внешнюю языковую модель из любого приложения.

Пользователь выделяет текст в любом окне Windows, нажимает настроенную глобальную горячую клавишу, выбирает заранее созданную задачу, после чего программа отправляет выделенный текст во внешнюю модель вместе с инструкцией этой задачи. Полученный ответ либо автоматически вставляется обратно в активное приложение, либо показывается в отдельном окне с возможностью продолжить диалог.

## Запуск и поведение в системе

- Программа запускается в одном экземпляре. Если один экземпляр уже работает, второй не открывается и показывает предупреждение.
- После запуска основное окно не отображается автоматически.
- Программа остается работать в области уведомлений Windows.
- Через значок в области уведомлений можно открыть окно настроек или полностью закрыть программу.
- Закрытие окна настроек не завершает программу, а скрывает окно.

## Основные настройки

Пользователь может настроить:

- адрес внешнего сервиса языковой модели;
- ключ доступа к внешнему сервису;
- модель по умолчанию;
- прокси-сервер для сетевых запросов;
- глобальную горячую клавишу для вызова меню задач;
- лимит символов, которые будут отправлены в модель из выделенного текста или сообщения пользователя.

Если лимит символов равен нулю, текст отправляется без ограничения.

Настройки сохраняются автоматически при изменении. При следующем запуске программа восстанавливает сохраненные настройки. Если сохраненных настроек нет или они повреждены, программа создает настройки по умолчанию.

По умолчанию:

- горячая клавиша: `Ctrl+Shift+Space`;
- лимит отправляемого текста: 1000 символов;
- создана одна задача `Explain`, которая кратко объясняет выделенный текст и показывает ответ в отдельном окне.

## Импорт и экспорт настроек

- Пользователь может экспортировать все настройки и список задач во внешний файл.
- Пользователь может импортировать настройки и задачи из ранее сохраненного файла.
- При ошибке импорта или экспорта программа показывает предупреждение.

## Работа с моделями

- В настройках можно выбрать модель по умолчанию.
- Для каждой задачи можно выбрать отдельную модель.
- Если в задаче не выбрана отдельная модель, используется модель по умолчанию.
- Список моделей загружается из настроенного внешнего сервиса по запросу пользователя при открытии выбора модели.
- В списке моделей есть поиск.
- Если загрузка списка моделей не удалась, программа показывает предупреждение.

## Задачи

Пользователь может создать несколько задач. Каждая задача содержит:

- название, которое показывается в меню задач;
- инструкцию для модели;
- выбранную модель или использование модели по умолчанию;
- максимальный размер ответа;
- уровень вариативности ответа;
- режим обработки ответа.

Доступны два режима обработки ответа:

- `Insert Answer`: программа вставляет ответ модели обратно в активное приложение;
- `Show Answer in Window`: программа открывает отдельное окно с ответом и продолжением диалога.

Пользователь может:

- добавлять новые задачи;
- удалять задачи с подтверждением;
- менять порядок задач;
- редактировать параметры задач;
- оставлять задачу без названия, тогда она отображается как `<Unnamed>`.

## Вызов меню задач

- Меню задач вызывается глобальной горячей клавишей и работает поверх других окон.
- Меню появляется рядом с текущей позицией курсора мыши.
- Если рядом с курсором недостаточно места, меню сдвигается так, чтобы остаться в видимой области экрана.
- Меню не забирает фокус у текущего приложения.
- Если задач нет, в меню показывается сообщение `No tasks configured`.
- Если меню уже открыто и горячая клавиша нажата снова, старое меню закрывается и создается новое.

В меню задач можно:

- выбрать задачу мышью;
- перемещаться по задачам клавишами `Tab`, стрелками вниз/вправо и вверх/влево;
- запустить выбранную задачу клавишами `Enter` или `Space`;
- закрыть меню клавишей `Esc`;
- закрыть меню кликом вне его области.

## Захват выделенного текста

- После выбора задачи программа пытается получить текст, выделенный в активном приложении.
- Для этого она использует стандартное действие копирования Windows.
- Если выделенный текст не найден, задача не выполняется.
- В режиме показа ответа в отдельном окне исходное содержимое буфера обмена восстанавливается после захвата выделенного текста.
- В режиме вставки ответа буфер обмена используется для последующей вставки результата.

## Режим вставки ответа

В режиме `Insert Answer` программа:

1. Скрывает меню задач.
2. Показывает небольшой индикатор загрузки рядом с курсором.
3. Отправляет в модель инструкцию задачи и выделенный текст.
4. При успешном ответе помещает ответ в буфер обмена.
5. Вставляет ответ в активное приложение стандартным действием вставки Windows.
6. Закрывает индикатор загрузки.

Индикатор загрузки следует за курсором и отображает текст `Loading` с меняющимся количеством точек.

Если запрос завершился ошибкой, программа закрывает индикатор загрузки и показывает предупреждение.

## Режим окна ответа

В режиме `Show Answer in Window` программа:

1. Скрывает меню задач.
2. Получает выделенный текст.
3. Открывает отдельное окно ответа.
4. Отправляет в модель инструкцию задачи и выделенный текст.
5. Показывает ответ по мере поступления.

Окно ответа поддерживает:

- отображение форматированного текста ответа;
- открытие внешних ссылок из ответа;
- продолжение диалога с моделью;
- сохранение истории сообщений в рамках открытого окна;
- поле ввода для уточняющих сообщений;
- отправку сообщения кнопкой или клавишей `Enter`;
- перенос строки в поле ввода через `Shift+Enter`;
- остановку текущего запроса;
- автоматическое изменение высоты поля ввода от 1 до 5 строк;
- автоматическую прокрутку вниз, если пользователь находился внизу ответа;
- изменение масштаба ответа через `Ctrl` + колесо мыши.

Первый запрос в окне содержит инструкцию задачи и выделенный текст. Последующие сообщения пользователя отправляются вместе с историей текущего диалога.

Если запрос в окне ответа завершается ошибкой, ошибка добавляется в область ответа как сообщение.

При закрытии окна текущий запрос отменяется.

## Сохранение параметров окна ответа

Для каждой задачи программа запоминает:

- ширину окна ответа;
- высоту окна ответа;
- масштаб текста ответа.

Эти параметры обновляются при изменении размера окна или масштаба и сохраняются для последующих запусков этой задачи.

## Отправка запросов во внешнюю модель

При выполнении задачи программа отправляет:

- инструкцию задачи как системное сообщение;
- выделенный текст или сообщение пользователя как пользовательское сообщение;
- выбранную модель, если она задана;
- максимальный размер ответа;
- уровень вариативности ответа.

Для режима вставки ответ ожидается целиком. Для режима окна ответа ответ принимается потоково и отображается постепенно.

Если внешний сервис временно отвечает ошибкой перегрузки или недоступности, программа повторяет запрос ограниченное число раз. Если сервис сообщает рекомендуемое время ожидания перед повтором, программа учитывает его.

Если ответ внешнего сервиса не содержит текста помощника, программа считает это ошибкой.

## Обработка ошибок

Программа показывает предупреждения при следующих ситуациях:

- попытка запустить второй экземпляр программы;
- ошибка получения списка моделей;
- ошибка запроса к модели в режиме вставки;
- ошибка импорта настроек;
- ошибка экспорта настроек.

В окне ответа ошибки запроса отображаются внутри самого окна ответа.
